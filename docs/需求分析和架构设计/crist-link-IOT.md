## CristLink-IoT 需求分析与架构设计（初稿）

### 一、核心设计思想与关键性能指标

CristLink-IoT：本项目，致力于构建一个基于`GO语言`生态的**分布式**、**高并发**的物联网接入平台。以支撑百万级（最低标准）设备接入与实时数据处理。

#### 1.1 核心设计思想

- **高并发接入**：采用`gnet`框架构建设备接入层，利用其`Reactor`模式与`epoll/kqueue`机制，实现单机十万级长链接的稳定支持。
- **微服务治理**：选用`Kratos`作为微服务框架，通过`Protobuf`强制接口定义，集成OpenTelemetry实现全链路追踪，保障服务间的“无缝对话”。
- **消息有序流转**：在`Kafka`中，以`DeviceID`为Key进行哈希路由，确保同一设备的消息进入到同一`Partiton`，从而保证分区内FIFO顺序，满足99%业务场景需求。
- **可靠运行保障**：实施30秒优雅退出流程，结合`sync.Pool`对象复用与`context.Context`取消信号传播，有效控制海量协程的内存消耗与生命周期。
